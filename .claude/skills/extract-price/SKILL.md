# extract-price

Скилл извлекает цену одного товара с его страницы по URL и возвращает строго
структурированный объект. Никакого свободного текста в ответе — только объект
контракта (JSON).

## Контракт

**Вход:** ровно один URL страницы товара (строка).

**Выход:** объект следующей формы:

```json
{
  "regular_price": 1299.0,
  "sale_price": 999.0,
  "has_credit": true
}
```

| Поле            | Тип             | Значение                                                                 |
| --------------- | --------------- | ------------------------------------------------------------------------ |
| `regular_price` | number          | Обычная (базовая) цена товара, число без валюты и пробелов.              |
| `sale_price`    | number \| null  | Цена по скидке. Если скидки нет — `null`.                                |
| `has_credit`    | boolean         | `true`, если на странице доступна рассрочка/кредит; иначе `false`.       |

Правила согласованности:
- Если на странице только одна цена — это `regular_price`, а `sale_price = null`.
- Если есть зачёркнутая старая цена и текущая сниженная — старая идёт в
  `regular_price`, текущая (сниженная) — в `sale_price`.
- `regular_price` не может быть `null` при успешном извлечении.

## Логика извлечения

1. **Получить содержимое страницы.**
   - Загрузи HTML страницы по URL (инструмент `WebFetch`, а если нужен «сырой»
     HTML/JS-рендер — браузер через Playwright/Chromium).
   - Многие магазины отдают цену в структурированных данных — проверь их в
     первую очередь, они надёжнее вёрстки:
     - JSON-LD: блоки `<script type="application/ld+json">` с
       `"@type": "Product"` → `offers.price`, `offers.priceCurrency`,
       `offers.availability`.
     - Микроразметка `itemprop="price"` / `meta[itemprop="price"]`.
     - OpenGraph: `meta[property="product:price:amount"]`,
       `meta[property="og:price:amount"]`.

2. **Где искать цену в вёрстке (fallback).**
   - CSS-селекторы и атрибуты, типичные для карточек:
     `[class*="price"]`, `[data-test*="price"]`, `[data-widget*="price"]`,
     `[itemprop="price"]`.
   - Текущая (актуальная) цена обычно крупнее и без зачёркивания; старая цена —
     зачёркнутая (`<del>`, `[class*="old"]`, `[class*="strike"]`,
     `text-decoration: line-through`).
   - Рассрочка/кредит: ищи слова «рассрочка», «в кредит», «Ozon Рассрочка»,
     «Сплит», «X ₽ / мес», «частями», «Podeli» и т.п. → это признак
     `has_credit = true`.

3. **Нормализация числа и валюты.**
   - Убери валютные символы и коды (`₽`, `руб`, `руб.`, `RUB`, `p.`).
   - Убери разделители тысяч: пробелы, неразрывные пробелы (` `),
     тонкие пробелы (` `), апострофы.
   - Приведи десятичный разделитель к точке (`,` → `.`), затем — к `number`.
   - Пример: `"1 299,00 ₽"` → `1299.0`; `"999 ₽"` → `999.0`.

4. **Если цена не найдена.**
   - Верни объект с `regular_price: null`, `sale_price: null`,
     `has_credit: false` и добавь поле `"error"` с кратким пояснением
     (например, `"price_not_found"` или `"page_requires_auth"`).
   - Не выдумывай цену и не возвращай свободный текст вместо объекта.

## Формат ответа

Верни **только** объект контракта (валидный JSON), без пояснений вокруг него.
Пример успеха:

```json
{ "regular_price": 1299.0, "sale_price": 999.0, "has_credit": true }
```

Пример, когда скидки и рассрочки нет:

```json
{ "regular_price": 549.0, "sale_price": null, "has_credit": false }
```

Пример, когда цена не найдена:

```json
{ "regular_price": null, "sale_price": null, "has_credit": false, "error": "price_not_found" }
```
