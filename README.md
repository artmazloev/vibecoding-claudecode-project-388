# Price Tracker — трекер цен на Claude-скиллах

Инструмент отслеживания цен товаров, собранный в подходе вайбкодинга: вся логика
живёт в скиллах Claude Code, а не в обычном коде. За один запуск трекер снимает
цены по списку товаров, сохраняет прогон в отдельный приватный репозиторий,
сравнивает со вчерашним прогоном и присылает в Telegram только значимые
изменения.

## Как это работает (полный цикл)

```
extract-price ──► tracker ──► tracker-data (история) ──► diff ──► Telegram
   (1 URL)        (список)      YYYY-MM-DD.json         (значимое)   (сводка)
```

1. **Извлечение цены** — скилл `extract-price` по одному URL карточки товара
   возвращает объект контракта `{ regular_price, sale_price, has_credit }`
   (обычная цена; цена по скидке или `null`; признак рассрочки). Нормализует
   число и валюту, отдаёт `null` при ошибке — не свободный текст.
2. **Сбор таблицы** — скилл `tracker` за один запуск обходит зашитый список
   URL, вызывает `extract-price` для каждого и собирает единую таблицу прогона.
3. **Сохранение прогонов** — свежий прогон сохраняется через GitHub MCP в
   отдельный приватный репозиторий `tracker-data` как файл `YYYY-MM-DD.json`
   (один прогон в день), без локального `git clone`.
4. **Сравнение с прошлым** — `tracker` читает предыдущий прогон из
   `tracker-data`, считает diff по `url` и оставляет только значимые изменения
   по правилам из `KNOWLEDGE.md`.
5. **Доставка в Telegram** — значимые изменения приходят отдельным сообщением
   через `send.py`; если значимых изменений нет — «Значимых изменений цен нет».
   Формулировку сводки делегирует дешёвому sub-agent'у `tg-summary` (Haiku).

## Структура репозитория

```
.claude/skills/
├── extract-price/SKILL.md   # один URL → { regular_price, sale_price, has_credit }
└── tracker/SKILL.md         # обход списка URL, таблица, хранение, diff, Telegram
.claude/agents/
└── tg-summary.md            # оформление TG-сводки на модели Haiku (опционально)
KNOWLEDGE.md                 # правила значимости изменений цен
send.py                      # отправка сообщения в Telegram (только stdlib, urllib)
.env.example                 # шаблон секретов (.env в .gitignore, не коммитится)
```

История прогонов хранится **отдельно** — в приватном репозитории `tracker-data`:

```
tracker-data/
└── YYYY-MM-DD.json          # один прогон в день (таблица прогона в JSON)
```

## Правила значимости (KNOWLEDGE.md)

Из diff отбрасывается шум, остаётся только то, о чём стоит уведомлять:

- **Цена** — изменение значимо при относительном сдвиге ≥ 1%; меньше — шум.
- **Скидка** — появление (`null → число`) или исчезновение значимо всегда;
  изменение размера — по порогу.
- **Рассрочка** — любое изменение `has_credit` значимо.
- **Состав списка** — новый товар (`added`) или пропавший (`removed`) значимы.
- **Первый прогон** — сравнивать не с чем, сохраняется снимок цен.

## Настройка Telegram

1. Создать бота в [@BotFather](https://t.me/botfather) → `/newbot` → получить токен.
2. Узнать свой chat id ([@userinfobot](https://t.me/userinfobot)).
3. Отправить своему боту любое сообщение (иначе бот не сможет написать первым).
4. Положить секреты (в репозиторий их коммитить нельзя):
   - локально — в файл `.env` (см. `.env.example`, `.env` уже в `.gitignore`);
   - в облаке/CI — в переменные окружения `TELEGRAM_BOT_TOKEN` и
     `TELEGRAM_CHAT_ID` (например, GitHub Actions → Secrets).

Проверка доставки отдельно:

```bash
python3 send.py "тест"   # ожидаемо: вывод "Отправлено." и сообщение в Telegram
```

## Запуск

Трекер запускается одной фразой в Claude Code — список URL и вся логика зашиты в
скилл:

```
запусти tracker
```

За запуск он снимет цены по списку, сохранит прогон в `tracker-data`, сравнит с
предыдущим и пришлёт значимые изменения в Telegram.

## Автономный запуск по расписанию (GitHub Actions)

Помимо запуска через Claude Code, тот же полный цикл можно гонять автономно — без
Claude и MCP — через GitHub Actions:

- `automation/tracker_run.py` — самостоятельная реализация цикла на чистом
  Python (только stdlib): снятие цен, таблица, чтение прошлого прогона из
  `tracker-data` через GitHub API, diff по правилам `KNOWLEDGE.md`, сохранение
  прогона и отправка сводки через `send.py`.
- `.github/workflows/tracker.yml` — запускает скрипт ежедневно (cron) и по
  кнопке (workflow_dispatch).

Секреты (репозиторий → Settings → Secrets and variables → Actions):

| Секрет | Назначение |
| ------ | ---------- |
| `TELEGRAM_BOT_TOKEN`, `TELEGRAM_CHAT_ID` | отправка в Telegram |
| `GH_DATA_TOKEN` | PAT с доступом на запись в приватный `tracker-data` (встроенного `GITHUB_TOKEN` недостаточно — он ограничен текущим репозиторием) |

## Технические заметки

- `send.py` использует только стандартную библиотеку Python (`urllib`) — ставить
  ничего не нужно.
- Запись прогонов в `tracker-data` идёт через **GitHub MCP** — Claude Code
  коммитит файл напрямую, без клонирования репозитория.
- Оформление сводки вынесено в sub-agent на **Haiku**: основная модель — логика
  трекера, дешёвая — только текст сводки.
